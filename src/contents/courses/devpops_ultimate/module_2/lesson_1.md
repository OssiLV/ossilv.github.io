Trong khóa học này, bạn có thể lựa chọn sử dụng **giao diện dòng lệnh (CLI)** hoặc **giao diện người dùng (UI)** để làm việc với Git. Tuy nhiên, trong khóa học này, chúng ta sẽ sử dụng **dòng lệnh (CLI)**. Vì sao lại chọn CLI?

- Git là một công cụ mã nguồn mở, có thể cài đặt trên Windows, macOS, Unix và Linux.
- Nếu bạn sử dụng giao diện UI, nó thường chỉ hỗ trợ tốt trên Windows.
- Nhưng nếu bạn là người dùng macOS hoặc Linux thì giao diện dòng lệnh là lựa chọn linh hoạt và phù hợp hơn.

Vì vậy, chúng ta sẽ làm việc với **Git Bash** hoặc **Git CLI** trong suốt khóa học này.

---

## Làm việc với CLI – Các lệnh Git

Để làm việc hiệu quả, bạn cần học và sử dụng một số **lệnh Git cơ bản**. Các lệnh Git có thể trông khá giống nhau, nhưng chức năng và hành vi của chúng có thể khác nhau tùy thuộc vào từng tình huống cụ thể.

Chúng ta sẽ:

- So sánh các lệnh Git tương tự nhau để hiểu rõ hơn cách chúng hoạt động.
- Giúp bạn lựa chọn lệnh phù hợp nhất trong từng trường hợp cụ thể.

---

## Git và vai trò trong DevOps

Hiện nay, Git là một phần **thiết yếu** trong DevOps. Dù bạn là kỹ sư DevOps hay kỹ sư phần mềm, chắc chắn bạn sẽ làm việc với Git trong dự án của mình.

Vì vậy, bạn cần hiểu rõ:

- Git hoạt động như thế nào.
- Tại sao Git lại quan trọng đối với DevOps.
- Cách Git giúp bạn quản lý mã nguồn, theo dõi thay đổi và phối hợp làm việc nhóm hiệu quả.

---

## Giới thiệu về GitHub

Chúng ta cũng sẽ tìm hiểu về **GitHub**:

- Git là một **công cụ phía client** cài đặt trên máy tính của bạn.
- GitHub là **máy chủ từ xa**, nơi bạn có thể đồng bộ mã nguồn từ máy cá nhân lên một máy chủ đám mây.
- GitHub giúp bạn cộng tác, quản lý phiên bản và chia sẻ mã nguồn với đồng đội hoặc cộng đồng.

Chúng ta sẽ học cách **tích hợp Git với GitHub** và sử dụng GitHub trong quy trình phát triển phần mềm hiện đại.
